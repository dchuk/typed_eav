<objective>
Ship two of the five Phase 5 field types — Image and File — using a lazy Active Storage soft-detect strategy that mirrors the existing acts_as_tenant precedent. After this plan: the gem can be required in a Rails app WITHOUT Active Storage and Image/File constants exist (autoloaded) but raise NotImplementedError on first use; the gem can be required in a Rails app WITH Active Storage and TypedEAV::Value gains a `:attachment` has_one_attached association registered at engine boot. The new `:on_image_attached` hook follows the Phase 3 hook contract pattern and fires from after_commit on TypedEAV::Value when an Image-field-typed Value gains an attachment.

This plan does NOT modify the gemspec to add Active Storage as a hard dependency. The decision (Gating Decision 1 from RESEARCH.md) is locked: lazy soft-detect via `defined?(::ActiveStorage::Blob)` is the chosen approach. The dummy app for the test suite DOES load Active Storage (so the integration specs can attach blobs and validate the full path). A new `active_storage_soft_detect_spec.rb` validates BOTH branches of the soft-detect: the loaded path (against the dummy app's loaded Active Storage) and the unloaded path (via a testable seam — a class method `TypedEAV::Engine.register_attachment_associations!` extracted from the after_initialize block, which can be called against a stubbed `defined?` predicate or a similar test seam).

After this plan ships, the `BUILTIN_FIELD_TYPES` hash holds 21 entries (19 from prior plans + Image + File). Plan 05-04 adds Reference for the final 22. The single shared `:attachment` association on TypedEAV::Value covers both Image and File — the field class identity (and the blob's content_type) discriminates at runtime. The `on_image_attached` hook is image-specific by ROADMAP design; File has no parallel hook (apps that need a generic file-attached signal use `on_value_change` from Phase 3).
</objective>

Locked decisions binding on this plan:

- **Gating Decision 1 RESOLVED:** LAZY SOFT-DETECT. No hard dependency in the gemspec. has_one_attached wrapped in `if defined?(::ActiveStorage::Blob)` inside config.after_initialize. Field::Image#cast and Field::File#cast raise NotImplementedError on first call when Active Storage is absent. The acts_as_tenant precedent (Config DEFAULT_SCOPE_RESOLVER) is the established gem-wide pattern.
- **Single attachment association:** `TypedEAV::Value.has_one_attached :attachment` (one association, NOT two). Image vs File distinction is by `value.field.is_a?(Field::Image)` at runtime, not by association name. RESEARCH §Risk 3 documents the rationale.
- **Storage column for Image/File:** `string_value` (signed_id portability). Choosing `string_value` over `integer_value` insulates the gem from Active Storage's internal blob-id format choices and aligns with the read-side `value.value → signed_id String` shape.
- **on_image_attached fires Image-only:** ROADMAP wording is image-specific. File-attached has no parallel hook; consumers use on_value_change (Phase 3) or subscribe to ActiveSupport::Notifications directly for file-specific signals.
- **after_commit dispatch (NOT ActiveSupport::Notifications):** Notifications ordering is fragile. The after_commit path is consistent with Phase 3's on_value_change and Phase 4's versioning subscriber.
- **Lazy class load:** Image/File constants exist (autoloaded) even without Active Storage. The raise happens on `cast` invocation. Rationale: gem consumers inspecting `TypedEAV::Field::Image.value_column` shouldn't be forced to load Active Storage just to read the constant.

Pattern conformance:
- `defined?(::ConstantName)` soft-detect (config.rb DEFAULT_SCOPE_RESOLVER pattern).
- `value_column :col_name` declaration (column_mapping.rb pattern).
- `operators(...)` explicit narrowing (Boolean, Color, Select precedent — narrows from a default).
- `store_accessor :options, :keys` for type-specific config (PATTERNS.md §3).
- Hand-rolled Config accessor for `on_image_attached` (matches scope_resolver / versioning / on_value_change idiom).
- `config.after_initialize` block in engine.rb for runtime registration (Phase 4 versioning subscriber precedent — engine.rb already has one such block).
- Testable seam for the engine initializer (Phase 4 introduced `Versioning.register_if_enabled` so the boot logic could be tested without rebooting Rails — same pattern here).

# frozen_string_literal: true

module TypedEAV
  module Field
    # Active Storage-backed field type. Stores ActiveStorage::Blob signed_id
    # in string_value. The attachment association lives on TypedEAV::Value
    # (has_one_attached :attachment, registered conditionally in Engine
    # config.after_initialize when ActiveStorage::Blob is defined).
    #
    # Soft-detect contract (Phase 5 Gating Decision 1): the gem does NOT
    # hard-depend on Active Storage. When Active Storage is absent at
    # runtime, this class still loads (Zeitwerk autoloads it), but #cast
    # raises NotImplementedError with an actionable install message on
    # first invocation.
    class Image < Base
      value_column :string_value

      operators :eq, :is_null, :is_not_null

      store_accessor :options, :allowed_content_types, :max_size_bytes

      # Cast contract: Hash/Blob/String → signed_id String;
      # nil/blank → nil; raw IO → invalid (apps must call
      # value.attachment.attach(io) directly).
      def cast(raw)
        unless defined?(::ActiveStorage::Blob)
          raise NotImplementedError,
                "TypedEAV::Field::Image requires Active Storage. " \
                "Add `gem 'activestorage'` to your Gemfile."
        end

        return [nil, false] if raw.nil? || (raw.respond_to?(:empty?) && raw.empty?)
        return [raw.signed_id, false] if raw.is_a?(::ActiveStorage::Blob)
        return [raw, false] if raw.is_a?(String)

        [nil, true]
      end

      def validate_typed_value(record, val)
        return if val.nil?
        return unless defined?(::ActiveStorage::Blob)

        blob = ::ActiveStorage::Blob.find_signed(val)
        if blob.nil?
          record.errors.add(:value, :invalid)
          return
        end

        if allowed_content_types.present? && !content_type_matches?(blob.content_type)
          record.errors.add(:value, "must be one of #{Array(allowed_content_types).join(', ')}")
        end

        if max_size_bytes.present? && blob.byte_size > max_size_bytes.to_i
          record.errors.add(:value, "exceeds max size #{max_size_bytes} bytes")
        end
      end

      private

      def content_type_matches?(blob_content_type)
        Array(allowed_content_types).any? do |pattern|
          if pattern.end_with?("/*")
            blob_content_type&.start_with?(pattern.sub("/*", "/"))
          else
            blob_content_type == pattern
          end
        end
      end
    end
  end
end
