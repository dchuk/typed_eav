## What Was Built

- `Entity.bulk_set_typed_eav_values(records, values_by_field_name, version_grouping:)` — class-method bulk-write API in `lib/typed_eav/has_typed_eav.rb` `ClassQueryMethods` (line 536). Outer `ActiveRecord::Base.transaction` envelope with per-record `transaction(requires_new: true)` savepoint (the CONTEXT-locked structure preserved under every `version_grouping:` value — never relaxed to per-record top-level transactions). Per-record body routes through `record.typed_eav_attributes=` (reusing field-name resolution, `allowed_typed_eav_types` restriction, cross-tenant guards) then `record.save`; failures capture `record.errors.messages.transform_keys(&:to_s)` and raise `ActiveRecord::Rollback` to roll back only the savepoint. Result shape `{ successes: [record, ...], errors_by_record: { record => { "field_name" => [messages] } } }` with the record itself (NOT `record.id`) as the errors key for unsaved-record correctness.
- `version_grouping:` four-mode contract: `:default` sentinel resolves to `:per_record` when `Config.versioning` is on and `:none` when off (omitted-kwarg "just works" in both environments); explicit `:per_record` generates one UUID per record so all field changes for record X share that UUID; explicit `:per_field` generates one UUID per unique field name in `values_by_field_name` so all records that had field Y updated share that UUID; explicit `:none` is the no-op opt-out. Argument validation raises `ArgumentError` on unknown values and on explicit `:per_record`/`:per_field` with versioning off (omitting and `:none` are always valid).
- Snapshot mechanism for `version_group_id` propagation across the outer-transaction `after_commit` boundary: added transient `pending_version_group_id` `attr_accessor` on `TypedEAV::Value` (in-memory ivar mirroring the existing `@cast_was_invalid` pattern at value.rb:118 — NOT a DB column). Amended the Phase 4 versioning subscriber's `ValueVersion.create!` call to write `version_group_id: value.pending_version_group_id || context[:version_group_id]`, preferring the per-Value snapshot over the context Hash. Bulk write stamps the snapshot on each dirty Value object (`v.new_record? || v.changed?`) BEFORE `record.save` inside the per-record `with_context` block; the stamp persists across the outer-transaction `after_commit` boundary even after `with_context` has lexically unwound. The `with_context` push is retained as a belt-and-suspenders fallback.
- N+1 prevention: outer wrap in `ActiveRecord::Base.cache do` (catches non-write reads) PLUS an explicit thread-local memo (`Thread.current[:typed_eav_bulk_defs_memo]`) consulted by `InstanceMethods#typed_eav_defs_by_name` so identical `typed_eav_definitions` SELECTs across records sharing one partition tuple are served from the memo instead of issuing N identical queries. The memo is set in the bulk method's `begin` and cleared in `ensure` so it never escapes the bulk call (zero overhead outside a bulk operation; supports nested bulk calls). The thread-local fallback was added because AR's `cache do` query cache is invalidated by every write, so cache-do alone cannot keep field-definition reads N+1-free across the bulk loop — documented in the inline rationale at has_typed_eav.rb:603-624.
- Spec coverage at `spec/lib/typed_eav/bulk_operations_spec.rb` — 29 examples organized into 11 describe blocks: happy path (default kwarg, versioning off), partial failure (savepoint isolation), input validation (empty/nil/non-Hash/wrong-class/mixed-class), version_grouping argument validation, allowed_typed_eav_types restriction (Product), version_grouping when versioning is OFF (4 cases), version_grouping when versioning is ON (default, explicit :per_record, :per_field, :none, snapshot end-to-end, outer-transaction rollback under all 3 modes, per-record savepoint isolation under :per_record), N+1 prevention (single partition + cross-partition), errors_by_record value shape. All examples use existing factories (`:contact`, `:product`, `:integer_field`, `:text_field`); versioning specs use `:event_callbacks, :real_commits` metadata flags and explicitly re-register the Phase 4 subscriber per-example.
