So reading `TypedEAV.current_context` inside the subscriber at outer-transaction-commit time would observe an empty context. The fix is the SNAPSHOT mechanism documented above (Plan 01 adds `Value#pending_version_group_id`; THIS plan stamps it before save inside the active `with_context` block; the subscriber reads it off the Value object instead of off `current_context`). Single save per record (NOT one save per field — the in-memory snapshot per Value carries the per-field UUID through the after_commit chain even though all values commit together).
