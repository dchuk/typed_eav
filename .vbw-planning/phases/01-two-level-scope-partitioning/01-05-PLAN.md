<objective>
Wire the new `parent_scope` axis through the `has_typed_eav` macro, the resolver chain (`resolve_scope`), the query path (`where_typed_eav`, `with_field`, `typed_eav_definitions`), the collision-precedence helper (`definitions_by_name`), and the Value-side cross-axis write guard. After this plan, an entity model can declare `has_typed_eav scope_method: :tenant_id, parent_scope_method: :workspace_id` and queries / writes will respect the full triple.

This is the largest plan in the phase. It depends on plans 01–04: the migration provides the column; plan 02 provides the tuple-returning resolver; plans 03–04 provide the partition-aware `for_entity` scopes and the Field-level orphan-parent guard.

Per CONTEXT.md decisions:
- `parent_scope:` kwarg on `where_typed_eav` reuses the existing `UNSET_SCOPE` / `ALL_SCOPES` sentinels (no new constants).
- Multimap (`unscoped { }`) branch is atomic-bypass: drops both scope AND parent_scope predicates entirely. No predicate-level filtering inside `unscoped`.
- The resolver-callable contract is hard-breaking: custom resolvers returning bare scalars must update. The strict `ArgumentError` raise lives inside `TypedEAV.current_scope` (plan 02), BEFORE `normalize_scope` is applied — so by the time `resolve_scope` consumes the result, the shape is already validated as `nil | [a, b]`. `resolve_scope` does not re-check shape; that would be dead code.
- Configuration-error guard (this plan, task 1): `parent_scope_method:` without `scope_method:` raises `ArgumentError` at class-load time. This closes the silent-dead-letter edge case where a model declaring only parent_scope would have all queries short-circuit to globals.

Composite resolution semantics (Lead's discretion call):
- Single-scope branch: `Field::Base.for_entity(name, scope: s, parent_scope: ps)` — the [val, nil].uniq pattern in for_entity (plan 03) returns the union of (s, ps), (s, nil), (nil, nil) rows. The orphan-parent invariant guarantees no (nil, ps) rows exist.
- Multimap branch: `where(entity_type: name)` — unchanged. The OR-collapse at field_id level naturally OR's across all (scope, parent_scope) combinations.

Three-way precedence in `definitions_by_name`:
- Sort key: `[d.scope.nil? ? 0 : 1, d.parent_scope.nil? ? 0 : 1]` puts globals first, scope-only middle, full-triple last.
- `index_by(&:name)` keeps the LAST entry on duplicate keys (Rails convention via `Array#to_h`).
- Therefore most-specific wins, mirroring today's "scoped beats global" extended to "two-key beats one-key beats global".
</objective>

Update `resolve_scope` (was `resolve_scope(explicit)` at line 246) to take both halves and return a tuple OR ALL_SCOPES OR raise. New signature:

  private

  def resolve_scope(explicit_scope, explicit_parent_scope)
    # Inside `TypedEAV.unscoped { }` — atomic bypass, drops both predicates.
    return ALL_SCOPES if TypedEAV.unscoped?

    # Determine the explicit-overrides path. If EITHER kwarg was passed
    # explicitly (i.e., not UNSET_SCOPE), normalize what was given and skip
    # ambient resolution entirely. Mixing explicit + ambient resolution
    # within one call would be confusing; explicit wins for the whole tuple.
    explicit_given = !explicit_scope.equal?(UNSET_SCOPE) || !explicit_parent_scope.equal?(UNSET_SCOPE)

    if explicit_given
      s  = explicit_scope.equal?(UNSET_SCOPE) ? nil : TypedEAV.normalize_scope([explicit_scope, nil]).first
      ps = explicit_parent_scope.equal?(UNSET_SCOPE) ? nil : TypedEAV.normalize_scope([nil, explicit_parent_scope]).last
      # Orphan-parent invariant at the read layer: a request for parent_scope
      # without scope is dead-letter (no rows can match). Don't raise — just
      # narrow the predicate. Field-level validator (plan 03) prevents the
      # corresponding write.
      return [s, ps]
    end

    # Models without scope_method: don't see ambient state.
    # NOTE: a model with parent_scope_method: but no scope_method: is impossible
    # to construct — the macro-time guard in has_typed_eav (this plan, task 1)
    # raises ArgumentError at class-load time. If we get here, either both are
    # set or only scope_method is set, never only parent_scope_method.
    return [nil, nil] unless typed_eav_scope_method

    # Ambient via TypedEAV.current_scope — already validated as `nil | [a, b]`
    # by plan 02. The resolver-callable strict-contract raise (ArgumentError on
    # a bare-scalar resolver return) lives inside TypedEAV.current_scope, NOT
    # here — by the time current_scope returns, the shape is guaranteed. Do not
    # duplicate that check here; it would be dead code.
    resolved = TypedEAV.current_scope
    if resolved.nil?
      if TypedEAV.config.require_scope
        raise TypedEAV::ScopeRequired,
              "No ambient scope resolvable for #{name}. " \
              "Wrap the call in `TypedEAV.with_scope(value) { ... }`, " \
              "configure `TypedEAV.config.scope_resolver`, or use " \
              "`TypedEAV.unscoped { ... }` to deliberately bypass."
      end
      return [nil, nil]
    end

    # resolved is guaranteed [a, b] (length 2 Array) by current_scope's contract.
    resolved
  end

Note the signature change is internal — `resolve_scope` is private. Public callers (where_typed_eav, with_field, typed_eav_definitions) are updated in the next task.
