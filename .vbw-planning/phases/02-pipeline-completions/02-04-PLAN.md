<objective>
Add `Field#backfill_default!` for backfilling existing Value rows with the field's configured default. The method iterates entities of `field.entity_type` in batches of 1000, scoped to entities matching the field's partition (`(scope, parent_scope)`), and creates a Value row with the cast default for any entity that doesn't already have a non-nil typed value for this field.

The execution model is locked from 02-CONTEXT.md: `find_each(batch_size: 1000)` with per-batch transactions, synchronous by default, no built-in Active Job dispatch (the README documents the recipe for users who want async). Per-batch transactions make the operation recoverable mid-run.

- field.entity_type is a String (e.g., "Contact"); resolve to AR class via field.entity_type.constantize.
- field.entity_type.constantize.find_each(batch_size: 1000) is the iteration entry point.
- self.class.value_column returns the symbol of the typed column (e.g., :integer_value).
- field.default_value returns the already-cast default (or nil).
- Skip rule: a Value row WHERE field_id = field.id AND <value_column> IS NOT NULL exists for the entity → skip.
- Partition match: entity.typed_eav_scope == field.scope (when field.scope is non-nil); entity.typed_eav_parent_scope == field.parent_scope (when field.parent_scope is non-nil); both defined by has_typed_eav InstanceMethods (line 555-565ish).
- find_or_create_by!(entity: entity) is risky here because of the uniqueness validator on (entity_type, entity_id, field_id) and because we want explicit insert semantics. Better: query for existence with WHERE clause, INSERT if absent, UPDATE if present-but-nil.

For Value row creation, backfill passes explicit `value: field.default_value` — it does NOT use UNSET_VALUE because backfill knows the default and is not subject to "no value: kwarg given" semantics. This is BC-safe regardless of Plan 02-03 ordering: explicit `value: x` continues to store x in both pre-sentinel and post-sentinel code.

No existing find_each + per-batch-transaction pattern exists in the gem; this is the first such method. Comment density should explain WHY find_each over each (memory bound), WHY per-batch transactions (recoverability), WHY explicit value: (idempotence + skip-rule semantics).

Method body:

```ruby
# ── Backfill ──

# Backfills existing entities with this field's configured default value.
# Iterates entities of `entity_type`, scoped to the field's
# (scope, parent_scope) partition, in batches of 1000. Each batch runs
# inside its own transaction so a long-running backfill can be
# interrupted and resumed (re-run picks up where the prior run stopped
# because the skip rule re-checks each batch).
#
# Skip rule (per-record): skip when the entity already has a non-nil
# typed value for this field. A Value row whose typed column is nil is
# still a candidate for backfill — the skip rule is "non-nil typed
# column," not "Value row exists" (matches CONTEXT.md).
#
# Partition match: when field.scope is non-nil, the entity must respond
# to typed_eav_scope and the value must match field.scope (as String).
# When field.parent_scope is non-nil, same check for typed_eav_parent_scope.
# When field.scope is nil (global field), no scope filter — iterate all
# entities of entity_type.
#
# Synchronous by default. For async dispatch, define your own job:
#
#   class BackfillJob < ApplicationJob
#     def perform(field_id) = TypedEAV::Field::Base.find(field_id).backfill_default!
#   end
#   BackfillJob.perform_later(field.id)
#
# (Documented in README; not built-in to keep the gem dep-free.)
def backfill_default!
  # Short-circuit: nothing to backfill if no default configured. We
  # explicitly do NOT write nil rows — backfill is for propagating a
  # configured default, not for materializing empty Value rows.
  return if default_value.nil?

  entity_class = entity_type.constantize
  column = self.class.value_column

  entity_class.find_each(batch_size: 1000) do |entity|
    next unless partition_matches?(entity)

    # rubocop:disable Rails/TransactionExceptionHandler -- per-batch tx makes the backfill recoverable; we intentionally swallow nothing — let any exception abort and surface to the caller, who can re-run idempotently.
    ActiveRecord::Base.transaction(requires_new: true) do
      backfill_one(entity, column)
    end
    # rubocop:enable Rails/TransactionExceptionHandler
  end
end
```

(Adjust the rubocop disable cop name to match the project's actual style if `Rails/TransactionExceptionHandler` doesn't apply; the test is just to use the project's "disable with `--` justification" convention from CONVENTIONS.md.)

**Add private helpers** to the existing private block at end of class:

```ruby
# Partition-match check used by backfill_default!. Skips entities whose
# scope axis disagrees with the field's. The check is symmetric to the
# Value#validate_field_scope_matches_entity validator (which guards the
# write path) — backfill must not write Values that the validator would
# reject.
def partition_matches?(entity)
  return false unless entity_partition_axis_matches?(entity, :scope)
  return false unless entity_partition_axis_matches?(entity, :parent_scope)

  true
end

def entity_partition_axis_matches?(entity, axis)
  field_axis_value = public_send(axis) # field.scope or field.parent_scope
  return true if field_axis_value.blank? # global on this axis: any entity matches

  reader_method = "typed_eav_#{axis}".to_sym
  return false unless entity.respond_to?(reader_method)

  entity_value = entity.public_send(reader_method)
  return false if entity_value.nil?

  field_axis_value.to_s == entity_value.to_s
end

def backfill_one(entity, column)
  # Find an existing Value row (any state). The uniqueness validator
  # ensures at most one per (entity, field).
  existing = TypedEAV::Value
    .where(entity: entity, field_id: id)
    .first

  if existing.nil?
    # No row: create with explicit default. Pass value: explicitly so
    # we don't trigger the UNSET_VALUE sentinel path (backfill knows the
    # default; no need to re-resolve it).
    TypedEAV::Value.create!(entity: entity, field: self, value: default_value)
  elsif existing[column].nil?
    # Row exists but typed column is nil: update to default.
    existing.update!(value: default_value)
  end
  # else: row exists with non-nil value → skip (skip rule).
end
```
